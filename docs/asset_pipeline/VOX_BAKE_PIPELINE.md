# VOX Bake Pipeline Spec

## Intent

`.vox` files are the preferred source format for objects that need directional pixel-art rendering.

The runtime should not directly render `.vox` files in the normal 2D game view. Instead, `.vox` files are baked into sprites and metadata.

## Pipeline

```text
assets/voxels/source/<category>/<asset>.vox
        ↓ parse/inspect
content/metadata/vox_assets.ron
        ↓ bake
assets/generated/vox_bakes/<asset_id>/<facing>.png
assets/generated/vox_bakes/<asset_id>/shadow_<facing>.png
assets/generated/vox_bakes/<asset_id>/thumbnail.png
content/generated/vox_bakes/<asset_id>.bake.ron
        ↓ registry
editor/game uses generated sprites + masks + metadata
```

## Initial facing standard

Use 4 facings first:

- north
- east
- south
- west

Later support 8 facings if the camera/art direction requires it.

## Bake profile fields

| Field | Purpose |
|---|---|
| `camera_pitch_degrees` | Fixed orthographic pitch. |
| `yaw_steps` | Four-way or eight-way. |
| `pixel_scale` | Output pixel scale. |
| `outline_enabled` | Whether to draw pixel-art outline. |
| `shadow_enabled` | Whether to bake shadow layer. |
| `ambient_occlusion` | Optional fake AO. |
| `palette_policy` | Preserve source palette or remap. |
| `transparent_background` | Required for runtime sprites. |
| `trim_mode` | None, bounds, or anchor-preserving trim. |

## Metadata generated by bake

- output sprite dimensions;
- anchor/origin;
- footprint bounds;
- collision mask;
- interaction points;
- facing map;
- source hash;
- bake timestamp;
- bake profile id;
- validation messages.

## Editor features required

- `.vox` browser;
- parse status;
- source preview;
- dimensions and voxel count;
- palette view;
- bake button;
- rebake all button;
- missing bake warnings;
- directional preview turntable;
- footprint/collision editor;
- interaction point editor;
- open source folder button;
- help link.

## Runtime features required

- directional sprite selection based on camera yaw;
- placement footprint collision;
- interaction point query;
- draw order/sort origin;
- seasonal/damage variant fallback.

## Validation checklist

- Source `.vox` exists.
- Source `.vox` parses.
- Required facings are baked.
- Footprint exists.
- Collision mask exists when blocking.
- Anchor exists and is valid.
- Source hash matches current bake or editor warns stale bake.
