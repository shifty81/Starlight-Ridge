# Dynamic Voxelizer Legal Note

## Project rule

The Starlight Dynamic Voxel System is a native Starlight Ridge system.

It may be inspired by broad workflow categories such as:

- mesh-to-voxel conversion,
- skinned mesh pose sampling,
- voxel dissolve effects,
- voxel particle effects,
- bone-driven voxel characters,
- secondary spring-bone motion.

It must not copy code, shaders, assets, samples, documentation text, package structure, or proprietary implementation details from any Unity Asset Store product or third-party package.

## Dynamic Voxelizer reference

The Unity Dynamic Voxelizer package is useful as a broad category reference only.

Allowed:

- discuss general desired capabilities,
- identify broad user-facing workflow ideas,
- define Starlight-native requirements.

Not allowed:

- copying source code,
- copying shader graphs,
- copying serialized assets,
- copying demo content,
- decompiling or reverse-engineering package internals,
- reproducing proprietary implementation details.

## Implementation requirement

All implementation should be written from first principles for Starlight Ridge using the project's own architecture, data contracts, renderer, command runner, editor tabs, and external-tool bridge.

## Third-party reference assets

Third-party reference models may be used only under their own license terms and must retain attribution metadata in the repo.

Final Starlight Ridge character templates should remain original project assets.
