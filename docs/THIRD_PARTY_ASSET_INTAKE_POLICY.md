# Third-Party Asset Intake Policy

Phase 53b adds a reference intake pattern for third-party voxel assets.

## Rules

1. Every third-party asset must live under `content/third_party/<source>/<asset_id>/`.
2. Every third-party asset folder must include `ATTRIBUTION.md`.
3. The asset source, author, license, and source URL must be recorded.
4. The generator may use third-party assets for comparison, import testing, and reference only unless the asset is explicitly promoted for production use.
5. Any production use of CC BY content must retain attribution and indicate modifications where applicable.
6. Final Starlight Ridge base templates should remain original project assets.
